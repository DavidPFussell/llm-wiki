├── input

│ ├── config

│ │ ├── hyperparameters.json │ │ └── resourceConfig.json

3. Input (training) data will be connected here

│ └── data

│

└── <channel_name>

│

└── <input data>

├── model

│ ├── code

4. Your code will create model

│ └── <script files>

│

└── output

└── failure

5. Final model from /opt/ml/model will be saved to S3 at the end of training session, before the container is destroyed.
